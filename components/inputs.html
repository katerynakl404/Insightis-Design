Component

Input

Text, email, search, password, number. State priority: disabled > error > focus > hover > default (hover never overrides focus/error). Left side = passive icons; right side = active utility actions only.

Sizes

States (priority: disabled > error > focus > hover > default)

default (try it)
hover
focus (progressive ring)
error (beats focus + hover)
empty + placeholder
disabled · with placeholder
disabled · filled (.is-filled)

Focus border animates progressively (0 → 3px ring) — matching the error-ring treatment. Hover never overrides focus or error states.

With icon — left = passive / right = active

Input group (composer) — all states

default (empty — submit disabled until valid)
focused (ring on the group)
error
disabled

First composer's submit button stays disabled until the input passes type="email" validation (try typing a valid address).

HTML

<!-- Leading passive icon (left side only) -->
<div class="ins-input-wrap">
  <svg class="ins-input-wrap__icon ins-input-wrap__icon--leading" ...></svg>
  <input type="search" class="ins-input ins-input--with-leading">
</div>

<!-- Trailing ACTIVE clear button (right side only — interactive) -->
<div class="ins-input-wrap" data-sb-clearable>
  <input class="ins-input ins-input--with-trailing is-filled" value="...">
  <button class="ins-input-wrap__action" aria-label="Clear">
    <svg>...close (same as alerts)...</svg>
  </button>
</div>

<!-- Disabled variants -->
<input class="ins-input" value="" disabled>                   <!-- empty -->
<input class="ins-input is-filled" value="Acme Corp" disabled> <!-- filled, value stays legible -->

<!-- Input group (composer) — button disabled until input is valid -->
<div class="ins-input-group" data-sb-validate-email>
  <input type="email" class="ins-input">
  <button type="submit" class="ins-btn ins-btn--primary ins-btn--radius-md">Subscribe</button>
</div>
