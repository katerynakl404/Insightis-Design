Alert / Toast

Inline alerts for form-level messages. Floating toasts for transient feedback. Four semantic variants.

Inline — info / success / warning / error

Heads up
Your trial ends in 7 days.
Saved
Metric definition has been published.
Approaching limit
You've used 80% of your query quota.
Query failed
The source database is unreachable. Retry

Unified icon system: info = i-in-circle; success = check-in-circle; warning = exclamation-in-triangle; error = x-in-circle. Stroke 1.5, rounded caps, 20×20. Same glyphs reused across toasts, banners, form-group messages.

Compact toast (single-line ephemeral feedback)

Same component system, just the --compact modifier. No title, 16×16 icon, tight padding.

HTML

<div class="ins-alert ins-alert--success" role="status">
  <svg class="ins-alert__icon" ...></svg>
  <div>
    <div class="ins-alert__title">Saved</div>
    <div>Your changes were published.</div>
  </div>
  <button class="ins-alert__close" aria-label="Dismiss">...</button>
</div>

<!-- Variants -->
<div class="ins-alert ins-alert--info">...</div>
<div class="ins-alert ins-alert--warning">...</div>
<div class="ins-alert ins-alert--error" role="alert">...</div>

<!-- Floating toast (bottom-right on desktop, bottom-center on mobile) -->
<div class="ins-toast ins-alert ins-alert--success" role="status">
  ...
</div>

Accessibility

  • Info/success/warning: role="status" (polite)
  • Error: role="alert" (assertive — interrupts screen reader)
  • Dismiss button needs aria-label
