Motion

Two motion classes with distinct rules. State-communicating motion always runs; decorative motion is off on mobile + reduced-motion.

Duration + easing tokens

TokenValueUse
--ins-duration-fast120msMicro-interactions, color swaps
--ins-duration-base180msDefault transitions (hover/focus)
--ins-duration-slow320msModal open, drawer slide
--ins-duration-xslow400msPage-level transitions
--ins-easing-standardcubic-bezier(.4,0,.2,1)Most transitions
--ins-easing-acceleratecubic-bezier(.4,0,1,1)Exits, dismissals
--ins-easing-deceleratecubic-bezier(0,0,.2,1)Enters, reveals

Motion classes — the rule

.ins-motion-state · always on

Spinners, typing dots, toast enter/exit, skeleton shimmer, progress bars. Communicate UI state — stopping them is worse than animating. Under prefers-reduced-motion they slow to 2×, don't stop.

.ins-motion-decorative · off <768

Marquee, staggered fade-in, card hover-lift, parallax. No state info; purely feel. Disabled on mobile (<768px) and under prefers-reduced-motion.

Snippet

/* State-communicating */
.my-spinner { animation: ins-spin 800ms linear infinite; }

/* Decorative — disabled on mobile (base rule), enabled on ≥768 */
.my-marquee { animation: none; }
@media (min-width: 768px) {
  .my-marquee { animation: scroll 30s linear infinite; }
}

/* Reduced-motion (global) is in base.css */
