Inactive vs Disabled

Two distinct roles — always pick the one that matches the element's purpose.

Inactive (AA-compliant)

Available but not emphasized. Placeholders, timestamps, captions, meta, resting UI icons. Meets AA contrast.

  • --ins-text-inactive → body-size meta / placeholders
  • --ins-border-inactive → dividers, resting card borders
  • --ins-icon-inactive → leading icons in inputs, meta glyphs

Disabled (exempt from AA)

Unavailable / non-interactive. Readable enough to understand context, but deliberately muted. Exempt from WCAG 1.4.3.

  • --ins-text-disabled → disabled labels, disabled buttons
  • --ins-border-disabled → disabled input borders
  • --ins-icon-disabled → icons in disabled controls

Do / Don't

✓ Do

Use semantic tokens in components: var(--ins-text-body), var(--ins-surface-card). Use text-inactive only for disabled/placeholder copy.

✗ Don't

Don't reference primitives directly in components (var(--ins-color-gray-300)). Don't use text-inactive for regular prose — it fails AA on cards.

HTML snippet

<p class="ins-text-body">Body copy uses semantic text-body.</p>
<h2 class="ins-text-h2">Heading uses text-heading.</h2>
<div class="ins-card">
  <p class="ins-text-body">Card prose</p>
  <small style="color: var(--ins-text-inactive);">Timestamp (inactive role)</small>
</div>
