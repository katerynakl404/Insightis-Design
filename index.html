Design system

Insightis v1.0.0

Dark-mode marketing design system for Insightis. Two-tier tokens (primitives → semantics), WCAG AA-compliant, mobile-first, Geist typography. Built for the marketing site — landing, about, roadmap, chat, privacy, data-analytics-teams pages.

What's here

Every token, every component, every state rendered live. Click any sidebar entry to open its page. Every snippet is copy-paste ready.

LayerFilePurpose
Tokensassets/tokens.cssAll primitives + semantic aliases + legacy aliases. Single canonical file.
Baseassets/base.cssReset + @font-face (Geist / Geist Mono) + typography role classes + utilities.
Componentsassets/components.cssAll 32 components. Consumes semantic tokens only.
Storybookassets/storybook.css + assets/storybook.jsDocs-site chrome + interactivity. Do not ship to production.

Rules for developers

  • Use semantic tokens in components: var(--ins-text-body), not var(--ins-color-gray-300)
  • Use primitive tokens only inside tokens.css
  • Never inline raw #hex or rgba() — every color is a named token
  • Mobile-first: base styles target <480px; add @media (min-width: 768px) for tablet+
  • Animations: .ins-motion-decorative only runs ≥768px; .ins-motion-state always animates

What's NOT included

  • No light theme (dark-only scope)
  • No JS framework bindings (React/Vue/etc wrap these classes themselves)
  • No build pipeline — drop the CSS files in, no transpilation needed
  • No full page templates — use layout blocks to compose pages

Changes vs source HTML pages

Documented in UX_IMPROVEMENTS.md. Biggest shifts:

  • Heading font: Clash Display / Inter / Instrument → Geist everywhere
  • Mono font: Space Mono → Geist Mono
  • Primary buttons: gradient + glow → flat teal-600
  • Breakpoints: ad-hoc 600/900 → 480/768/1024/1280 (existing pages migrate per-touch)
  • Animations: full mobile → decorative off <768, state-communicating always on
